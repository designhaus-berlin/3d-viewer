<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iframe → parent page | postMessage von viewer zu demo-Seite; gleicher Origin (github.io) |
| parent page → iframe | send() ruft postMessage mit `*` als targetOrigin auf |
| browser → vendor/db-ux/ | Statische Assets — nur Download-Zeitpunkt ist riskant (CDN) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | postMessage send() mit `'*'` targetOrigin | accept | Viewer-Demo ist öffentlich; kein sensitives Payload; `'*'` ist Standardpraxis für same-domain embedding. Kommentar im Code weist auf konkreten Origin für Produktion hin. |
| T-02-02 | Information Disclosure | CDN-Download von DB UX Assets | mitigate | Assets werden einmalig von jsdelivr heruntergeladen und dann self-hosted. Kein laufender CDN-Request von Nutzern. Download in Task 1 durch Executor kontrolliert. |
| T-02-03 | Tampering | vendor/db-ux/ Dateien nachträglich verändert | accept | Kein Produktions-Backend; statische Files im Git-Repo. Integrität durch Git-History sichergestellt. |
| T-02-04 | Denial of Service | Iframe lädt Viewer mit großem Modell | accept | AE86 = 812 KB — unkritisch. Kein user-controlled model-Parameter auf der Demo-Seite. |
| T-02-05 | Elevation of Privilege | sandbox-Attribut am iframe | mitigate | iframe hat `sandbox="allow-scripts allow-same-origin allow-pointer-lock allow-popups"` — kein `allow-forms`, kein `allow-top-navigation`. Viewer kann nicht aus dem Sandbox-Kontext ausbrechen. |
</threat_model>
